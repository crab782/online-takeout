# 外卖系统 - 黑盒测试文档

## 测试目的
- 验证外卖系统的各项功能是否正常运行
- 确保系统在不同场景下的稳定性和可靠性
- 发现并修复潜在的功能缺陷和用户体验问题
- 确保系统符合用户需求和业务逻辑

## 测试环境要求

### 硬件要求
- 计算机：至少 4GB 内存，20GB 可用磁盘空间
- 网络：稳定的网络连接

### 软件要求
- 浏览器：Chrome 90+、Firefox 88+、Edge 90+
- 操作系统：Windows 10+、macOS 10.15+、Linux
- 分辨率：1024x768 及以上

### 系统环境
- 前端：Vue 2 应用，运行在 http://localhost:80
- 后端：Spring Boot 应用，运行在 http://localhost
- 数据库：MySQL

## 测试范围

### 1. 用户端功能
- 注册与登录
- 首页浏览与搜索
- 店铺详情与点餐
- 购物车管理
- 订单提交与支付
- 个人中心管理
- 订单历史与详情
- 地址管理

### 2. 商家端功能
- 登录与退出
- 数据看板查看
- 菜品管理（添加、编辑、删除）
- 套餐管理（添加、编辑、删除）
- 订单管理（查看、处理）
- 店铺管理
- 提现管理

### 3. 平台端功能
- 登录与退出
- 数据看板查看
- 店铺管理（审核、查看）
- 用户管理
- 订单管理
- 财务管理

## 测试步骤

### 通用测试步骤
1. 打开浏览器，访问系统URL
2. 按照测试用例的操作步骤执行
3. 记录执行结果
4. 验证结果是否符合预期
5. 如有异常，记录详细信息

## 通过/失败的判断标准

### 通过标准
- 功能按照预期正常运行
- 界面显示正确，无布局错乱
- 响应时间合理（页面加载时间 < 3秒）
- 数据处理正确，无数据丢失或错误
- 错误提示清晰明了

### 失败标准
- 功能无法正常运行
- 界面显示错误或布局错乱
- 响应时间过长（页面加载时间 > 5秒）
- 数据处理错误，出现数据丢失或错误
- 错误提示不清晰或无提示

## 测试注意事项

1. 测试前确保系统已经正确部署并运行
2. 测试过程中保持网络连接稳定
3. 测试时使用不同的浏览器进行兼容性测试
4. 测试过程中记录详细的测试步骤和结果
5. 遇到问题时，尝试重现并记录详细的错误信息
6. 测试完成后，整理测试结果并生成测试报告

## 测试终止条件

1. 所有测试用例执行完毕
2. 发现严重缺陷，需要修复后才能继续测试
3. 测试时间超过预定时间
4. 系统环境发生重大变化
